1 סקירת המערכת

פורטל VIP הוא אפליקציית אינטרנט חד-עמודית (SPA) הפועלת על 4 שלבים, נועדה ללקוחות האולפן של יקיר כהן. כל הנתונים עוברים דרך Make.com ולא מאוחסנים בקוד עצמו.

🔐

אפס מפתחות API בקוד. כל התקשורת עם Google Sheets, WhatsApp, ו-Claude AI עוברת דרך Make.com Webhooks בלבד. הכלי מחזיר טוקן אימות שנשמר ב-sessionStorage (TTL 24 שעות).

קובץ אחד מכיל 4 שלבים:

admin.html – פאנל ניהול

גישה ל-yoursite.com/admin.html – כניסה בסיסמה דרך Webhook.

  • סקירה: סטטיסטיקות (לקוחות, הזמנות, הכנסות)
  • לקוחות: טבלה עם חיפוש, סינון לפי שלב, פתיחת פרופיל
  • הזמנות: כרטיסי upsell ממתינים → סמן כטופל
  • יצוא CSV עם BOM לעברית ב-Excel
  • שליחת WhatsApp ישיר לכל לקוח

3 זרימת משתמש – איך זה עובד

1

לקוח נכנס ל-portal.html

מזין מספר טלפון ישראלי (05X-XXX-XXXX). הפורטל קורא ל-Webhook SEND_OTP.

2

Make.com שולח OTP ב-WhatsApp

הסצנריו ב-Make.com יוצר קוד 4 ספרות, שומר ב-Google Sheets, ושולח ב-WhatsApp ללקוח.

3

לקוח מזין קוד → אימות

הפורטל קורא ל-Webhook VERIFY_OTP. Make.com מחזיר {"success":true,"token":"abc123"}. הטוקן נשמר ב-sessionStorage ל-24 שעות.

4

שלב 2 – מניפסטו

לקרוא ולאשר. שלב ללא Webhook – מקומי בדפדפן בלבד.

5

שלב 3 – אונבורדינג

תמונה → UPLOAD_PHOTO (שמירה ב-Drive). הקלטת קול → TRANSCRIBE (Claude AI). שם אמן + ז'אנר → SAVE_PROFILE (שמירה ב-Sheets).

6

שלב 4 – דשבורד + Upsells

לקוח רואה WiFi, שיתוף חבר, ואפשרויות שדרוג. לחיצה על "הזמן" → Webhook ORDER.

✓

Admin מקבל הכל

כל הפרטים מופיעים ב-admin.html עם אפשרות לאשר הזמנות ולשלוח WhatsApp.

4 הגדרה ראשונית – Make.com Webhooks

⚠️

חובה לפני שיגור! כל ה-Webhooks מוגדרים כ-REPLACE_... בקוד. יש להחליף כל אחד ב-URL אמיתי מ-Make.com.

Webhooks שצריך ליצור ב-Make.com (portal.html)

שםמה לעשות בסצנריומה להחזיר
SEND_OTP מקבל phone. יוצר קוד 4 ספרות, שומר ב-Sheets, שולח ב-WhatsApp Business. {"success":true}
VERIFY_OTP מקבל phone, otp. בודק מול Sheets. אם נכון – מחזיר טוקן. {"success":true,"token":"abc..."}
UPLOAD_PHOTO מקבל phone, photo (Base64), filename. שומר ב-Google Drive. {"success":true}
TRANSCRIBE מקבל audio (Base64), phone. שולח ל-Claude API. מחזיר תמלול. {"transcription":"..."}
SAVE_PROFILE מקבל phone, artistName, genre, transcript. שומר ב-Sheets. {"success":true}
ORDER מקבל phone, items[]. שומר הזמנה, שולח WhatsApp לאדמין. {"success":true}
SHARE מקבל phone. רושם שיתוף. אופציונלי. {"success":true}

Webhooks שצריך ליצור ב-Make.com (admin.html)

שםמה לעשות בסצנריומה להחזיר
ADMIN_LOGIN מקבל password. בודק מול סיסמה שמוגדרת ידנית בסצנריו (ראה סעיף 5). {"success":true,"token":"admin-token-123"}
GET_CLIENTS שולף את כל הרשומות מ-Google Sheets. {"clients":[...]}
GET_ORDERS שולף הזמנות פתוחות מ-Sheets. {"orders":[...]}
DONE_ORDER מקבל orderId. מסמן הזמנה כטופלה ב-Sheets. {"success":true}
SEND_WA מקבל phone, name. רושם לוג. (הפתיחה עצמה היא ב-wa.me URL) {"success":true}

פתחו את portal.html בעורך טקסט (VS Code / Notepad++). חפשו REPLACE_ והחליפו:

5 סיסמת האדמין – איך מגדירים

🔒

הסיסמה לא מאוחסנת בקוד. זו החלטת אבטחה מכוונת – אם מישהו יראה את קוד המקור, לא יוכל לדעת את הסיסמה. היא מאוחסנת רק ב-Make.com Scenario.

כיצד להגדיר סיסמה לפאנל Admin

1

צרו Scenario חדש ב-Make.com

Webhook (Custom Webhook) → Router → HTTP Response

2

הוסיפו Router עם תנאי

תנאי: {{body.password}} = YOUR_CHOSEN_PASSWORD

אם נכון → החזר: {"success":true,"token":"VIP-ADMIN-2026"}

אם שגוי → החזר: {"success":false,"message":"סיסמה שגויה"}

3

העתיקו את URL ה-Webhook

הדביקו אותו ב-admin.html כ-ADMIN_LOGIN

✓

לכניסה לפאנל

גשו ל-yoursite.com/admin.html והזינו את הסיסמה שבחרתם ב-Make.com. Session נשמר 8 שעות.

💡

טיפ אבטחה: הטוקן שמוחזר ("VIP-ADMIN-2026") יכול להיות כל מחרוזת שתרצו. הוא נשלח עם כל קריאה ל-Admin Webhook כ-Header: X-VIP-Admin: VIP-ADMIN-2026. כך תוכלו לוודא בכל סצנריו ב-Make.com שהבקשה מגיעה מהפאנל המוסמך.

6 שימוש בפאנל Admin

סקירה כללית

כפתורפעולה
🔄 רענן (ראש הדף)שולף נתונים עדכניים מ-Sheets (מחק cache + טען מחדש)
יצוא CSVמוריד קובץ Excel עם כל הלקוחות + BOM לעברית

סינון לקוחות

  • לחצו על שם לקוח בטבלה → פרופיל מלא עם כפתור WhatsApp
  • סנן לפי שלב (הכל / שלב 1-4) + חיפוש חופשי
  • כפתור 📱 בעמודה האחרונה → פותח WhatsApp עם הודעה מוכנה

הזמנות (Upsells)

  • כרטיס הזמנה מציג: שם לקוח, מוצרים, סכום, תאריך
  • לחצו "✓ טופל" → מעדכן ב-Sheets + מוסיף ל-הכנסות
  • Badge צהוב בסיידבר מציג כמות הזמנות ממתינות

Session Admin

Session מחזיק 8 שעות ב-sessionStorage. לאחר 8 שעות ← מנוחל אוטומטית לדף כניסה.

7 העלאה לאתר (Deployment)

שלבי העלאה

  1. ודאו שהחלפתם את כל ה-REPLACE_ URLs ב-Make.com Webhook URLs אמיתיים
  2. שנו CONFIG.DEBUG = false בשני הקבצים (portal.html + admin.html)
  3. העלו את 4 הקבצים לשרת Apache עם SSL (HTTPS):
      portal.html | admin.html | .htaccess | robots.txt
  4. וודאו שה-.htaccess מופעל: AllowOverride All ב-apache2.conf
  5. בדקו את הכותרות: securityheaders.com
ℹ️

הגנה על admin.html: ניתן להוסיף ל-.htaccess הגבלת IP על admin.html כשכבת אבטחה נוספת. הוסיפו: <Files "admin.html"> Require ip YOUR_IP </Files>

הגדרת WiFi מ-URL

ניתן לשלוח ללקוח קישור ייחודי עם פרטי WiFi:

https://yoursite.com/portal.html?wifi=StudioVIP_5G|VIP2024!
                                              ↑ שם רשת     ↑ סיסמה

10 שאלות נפוצות

❓ לקוח לא קיבל OTP ב-WhatsApp

1. בדוק שה-Webhook SEND_OTP מוגדר נכון ב-Make.com ופעיל (לא Off). 2. בדוק ב-console של הדפדפן שאין שגיאת network. 3. ודא שמספר WhatsApp Business שלך אינו חסום.

❓ קוד OTP "שגוי" תמיד

ב-Webhook VERIFY_OTP – ודא שהסצנריו מחפש את הקוד בטבלה הנכונה ומחזיר {"success":true,"token":"..."} בדיוק (לא null, לא empty).

❓ תמונה לא עולה / גדולה מ-5MB

הגודל המקסימלי מוגדר ב-CONFIG.MAX_FILE = 5_000_000. שנה אם צריך. ב-Make.com – ודא שהסצנריו תומך ב-Base64 גדול (זה עלול לדרוש Plan גבוה יותר).

❓ כל הכפתורים נעולים (spinner רץ)

המשמעות היא שה-Webhook לא החזיר תשובה תוך 12 שניות (WH_TIMEOUT). בדוק ב-Make.com שהסצנריו מחזיר Response מיידי ולא "waiting for next run".

❓ Admin לא רואה לקוחות

ודא ש-Webhook GET_CLIENTS מחזיר מבנה נכון: {"clients":[{"phone":"05...","artistName":"...","step":4,...}]}. שמות שדות צריכים להיות זהים (camelCase).

❓ עברית מבולגנת ב-CSV

ודאו שפותחים את הקובץ ב-Excel עם encoding UTF-8 BOM. ה-BOM (\uFEFF) כבר מוסף אוטומטית בקוד.

❓ כיצד לשנות את טקסט המניפסטו?

פתחו portal.html, חפשו "אצלנו באולפן" – זה תחילת אזור המניפסטו ב-HTML (שלב 2).

❓ כיצד לשנות את תוכן ה-Upsells?

שני מקומות: 1. ה-HTML של הדשבורד (כרטיסי upsell). 2. האובייקט NAMES ב-orderUpsells() ב-JavaScript. שים לב: המחירים הוסרו מהמערכת – אין להציג ₪ ללקוחות שכבר שילמו.
